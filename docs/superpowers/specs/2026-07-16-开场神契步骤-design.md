# 开场捏角新增「神契」步骤 — 设计文档

日期：2026-07-16
状态：已确认

---

## 概述

在 `src/开场` 捏角流程中新增「神契」步骤，放在「行装功法」（第 4 步）和「属性」（原第 5 步）之间。玩家可从六位古神中择一缔结神契作为开局陪伴者，也可不选。

---

---

## 步骤流程变更

**原流程：**
出身(1) → 道途(2) → 宗门(3) → 行装功法(4) → 属性(5) → 出生点(6) → 总览(7)

**新流程：**
出身(1) → 道途(2) → 宗门(3) → 行装功法(4) → **神契(5)** → 属性(6) → 出生点(7) → 总览(8)

## 数据结构

### catalog/神契.ts（新建）

```typescript
export interface 神契条目 {
  名称: string;        // 古神名
  身份: string;        // 如「欲望之母」
  位阶: string;        // 上位神/下位神
  领域: string;        // 欲望·秩序 等
  神契技能: string;    // 技能简短名称
  神契描述: string;    // 技能详细描述
  描述: string;        // 古神一句话简介
  头像: string;        // 复用 avatarStore.ts 中 URL
}

export const 神契列表: 神契条目[] = [ /* 六位古神 */ ];
```

### draft.ts 新增

```typescript
const 已选神契 = ref<string>('');  // 空字符串 = 未选择
```

### 选中逻辑

- 选择 → `主角.神契装备 = 古神名`，draft 记录选择
- 取消 → `主角.神契装备 = ''`
- 换选 → 先清再设
- **不设置 `古神列表[名].收服 = true`**，收服留在游戏流程中自然发生

## 组件设计

### StepDivineContract.vue（新建）

- 卡片网格布局（2 列 × 3 行），与 StepShop 风格一致
- 每张卡片包含：**头像** → **名称** → **身份标签** → **位阶·领域** → **神契技能名** → **技能简述**
- 头像复用 `src/修仙状态栏/avatarStore.ts` 中的 `DEFAULT_AVATARS` URL
- 选中状态：金红边框发光 + 微微放大 + 左上角勾标记
- 底部提示「神契可选，不选亦可开局」

## 数据流

```
玩家选神契 → draft.已选神契 = 名
           → data.主角.神契装备 = 名
           → 存档携带已选神契
           → 总览展示神契信息
           → Prompt 读神契装备，开场叙事包含古神陪伴
```
