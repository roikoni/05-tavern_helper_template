# 战斗系统重置与美化升级设计

> 日期: 2026-06-26
> 范围: `src/黑色修士` 战斗系统、战斗格式、战斗美化、流派机制

## 一、目标

1. **战斗多样化**: 10 个修炼流派各有招牌机制，不同流派打出不同节奏与效果。
2. **战斗美化惊艳**: 从单一红色静态模板，升级为按流派/伤害类型/暴击大失败分化的动态美化，带动画与交互。
3. **`<combat>` 结构自由**: 放宽死板的 markdown 格式约束，改为固定字段标签 + 标签内文字自由。
4. **格式指导统一**: 世界书写一套格式规则指导 AI 输出，使输出稳定符合解析需要。

## 二、不改动的部分（保留骨架）

- 六维战力（力道/体魄/身法/灵力/神识/根骨）与修正值
- HP / MP / San / 洗脑值 公式
- 先手检定、D20 vs DC、武器骰/法术骰/精神骰
- 暴击（自然20）/ 大失败（自然1）/ 擦伤 / 落空
- 异常状态表（眩晕/沉默/灼烧/中毒/破甲/减速/定身/混乱/恐惧/精神污染）
- 克苏鲁精神系技能品级（低语/侵蚀/污染/支配/禁忌）
- 伤害计算公式（物理/法术/精神/技能）

这些是已验证的扎实骨架，流派机制叠加其上，不替换。

## 三、`<combat>` 新标签结构

### 设计原则

- **固定字段标签**: LLM 用固定标签标记字段，标签内文字自由。
- **字段可缺省**: 非本回合适用的字段可省（如落空时省 `<damage>`，无应对时省 `<response>`）。
- **顺序不强制**: 字段顺序可变，解析按标签名提取。
- **不靠 markdown 粗体匹配**: 摆脱"必须以 `**⚔️ ` 开头""必须有 `· ` 中点"等脆弱约束。

### 标签定义

| 标签 | 含义 | 是否必填 |
|---|---|---|
| `<turn>` | 回合标题（自由措辞，如"第3回合 — 奈亚的回合"） | 必填 |
| `<school>` | 流派与招牌状态（如"剑修 · 剑意 3→5"） | 可选 |
| `<action>` | 主动方行动 | 必填 |
| `<response>` | 被动方应对 | 可选 |
| `<roll>` | 检定过程与结果 | 必填 |
| `<damage>` | 伤害结算（落空时省） | 可选 |
| `<status>` | 状态变化（HP/MP/San 变化与当前值） | 必填 |

### 输出示例

```
<combat>
<turn>第3回合 — 奈亚的回合</turn>
<school>克苏鲁修 · San 侵蚀</school>
<action>奈亚发动 疯狂低语（消耗 MP 12 + 自身扣 San 3）</action>
<response>沈清辞选择神识抵御</response>
<roll>D20(17)+神识修正(6)=23 vs 神识抵御DC(18) → 命中</roll>
<damage>D8(6)+神识修正(6)-神识减免(3)=9点（精神/San）</damage>
<status>
奈亚 MP-12, San-3
沈清辞 San-9, 当前 HP:160/160 | MP:92/110 | San:81
</status>
</combat>
```

### 解析策略

- **正则（兜底）**: 只匹配 `<combat>...</combat>` 边界，包成带 `data-combat` 的容器 div，做轻量静态美化。不再死扣 em dash/中点/粗体。
- **脚本（增强）**: 读容器内文本，按 `<tag>` 提取字段，挂载 Vue 组件。脚本挂了 → 正则保底静态版，不会变成裸文本。

## 四、10 流派招牌机制

### 设计原则

- 每个流派 1 个核心机制 + 1 个战斗内状态（计数器/层数）。
- 状态写入 `<school>` 标签，战斗结束清零，**不进 MVU 长期变量**（不增加存档负担）。
- LLM 每回合读上一回合状态 → 算这一回合，负担可控。
- 凡人/无流派角色走纯通用规则，无招牌机制。
- 古神统一套"克苏鲁修"招牌机制（San 侵蚀）做基底，专属神契技能叠加其上。

### 流派机制总览

| 流派 | 战斗内状态 | 招牌机制 | 节奏改变 |
|---|---|---|---|
| 剑修 | 剑意 N | 剑技/普攻命中叠层，层数→剑技伤害乘数；换应对/被打断清零 | 鼓励连续进攻 |
| 法修 | 元素印（火/水/雷/风/土） | 法术命中叠元素印，两种异元素触发"反应"追加效果 | 元素搭配策略 |
| 体修 | 架势（攻/守/崩） | 切架势改检定倾向：守=减伤低伤、攻=反之、崩=下回合必暴击但本回合硬抗 | 节奏切换博弈 |
| 丹修 | 毒层 N / 药效 | 命中附毒叠层每回合结算毒伤；可服丹换临时强化但付 HP 代价 | 正面弱，补以续航+控场 |
| 阵修 | 阵眼 N | 行动布阵眼，达阈值触发阵法（困敌/增减益/范围伤） | 正面弱，补以区域控制+减益 |
| 符修 | 符×数量 | 预制符箓战斗中瞬发不耗 MP 但耗材料，可叠多符连发 | 正面弱，补以灵活+爆发 |
| 魂修 | 魂噬 N | 神识攻击叠层，达阈值可"夺魂"使对方一回合行动受限；自身易遭反噬 | 诡异控场，高风险高回报 |
| 魔修 | 心魔 N | 扣 HP 换伤害倍率与吸血，HP 越低伤越高；高心魔时检定不稳定（走火风险） | 赌命型 |
| 克苏鲁修 | 洗脑值/San | 精神攻击累积洗脑，使用者亦扣 San；古神沿用此基底 + 专属神契 | 双刃侵蚀 |
| 散修 | 应变 N | 可借鉴对手上一回合的应对方式一次，灵活但无专精上限 | 杂学应变 |

### 平衡性说明

丹修/阵修/符修在原作设定中"战力偏弱/布置耗时/材料消耗大"，招牌机制保留这些短板，但在其他维度补强：
- **丹修**: 正面伤害弱，但毒伤 DOT 提供续航，服丹可临时强化，控场能力强。
- **阵修**: 正面伤害弱且慢热，但阵法一旦成型提供强力区域控制与群体减益。
- **符修**: 正面持续输出弱（受材料限制），但瞬发不耗 MP 提供灵活性与爆发窗口。

## 五、世界书格式指导（替换原 战斗格式.txt）

世界书 `战斗格式.txt` 重写为**标签结构规范**：

- 说明 7 个标签的含义与必填/可选。
- 给出完整输出示例。
- 约束：`<combat>` 与 `</combat>` 各占一行；一次只结算一个回合；每个战斗叙事可生成一个回合。
- **不再要求** em dash、中点、粗体等具体格式符——标签内文字自由。
- 这套指导保证 AI 输出稳定符合正则/脚本解析需要。

## 六、战斗美化方案（正则兜底 + 脚本增强）

### 正则层（`正则\[美化]战斗结算.txt`）

- 匹配 `<combat>...</combat>` 边界。
- 包装为带 `data-combat` 属性的容器 div。
- 容器内做轻量静态美化（比现状略强，作为脚本挂掉时的保底）。
- 仍在 `仅格式显示` 作用，最大深度 3。

### 脚本层（新增 `src/战斗美化/` 脚本项目）

- 监听消息渲染事件，扫描楼层 DOM 找 `data-combat` 容器。
- 读容器内文本，按 `<tag>` 解析字段。
- 挂载 Vue 组件，负责：
  - **流派主题色**: 剑修银白、法修五行色、体修土黄、丹修青绿、阵修玄紫、符修朱砂、魂修幽蓝、魔修暗红、克苏鲁修诡紫、散修灰褐。
  - **动画**: 暴击闪光、大失败震屏、伤害飘字、状态条变化。
  - **交互**: 可折叠详情（点击展开检定细节）、跨回合状态追踪（读上一回合 `<school>` 状态）。
- 脚本以无沙盒 iframe 运行，`window.$` 作用于酒馆页面，参考 `.cursor/rules/脚本.mdc`。
- 关闭脚本时卸载组件与样式。

### index.yaml 注册

- 新增脚本库条目：`战斗美化`，`类型: 脚本`，`内容: import` 指向打包产物。
- 原 `[美化]战斗结算` 正则保留（兜底）。

## 九、风险与缓解

- **LLM 学不会新标签**: 风险中。缓解——格式指导给清晰示例 + 完整字段说明；标签语义直观。
- **脚本失败降级**: 缓解——正则兜底，脚本挂了出静态美化版而非裸文本。
- **流派机制 LLM 执行偏差**: 缓解——状态只跨回合不跨战斗，每回合读上一回合即可；机制模板统一格式。
- **多脚本相互干扰**: 这卡已挂 4 个脚本。缓解——战斗美化脚本写得偏保守，少依赖、强容错。
