# 炼娘术师 — 善恶剧情线 EJS 控制系统

## 概述

基于 MVU 变量 `user.善恶值`（范围 -100~100），用纯 EJS（世界书条目）实现善恶剧情线的条件触发。不依赖绿灯关键词，完全由 EJS 判断变量值来控制。

## 触发规则

| 条件 | 触发条目 | 说明 |
|---|---|---|
| 善恶值 ≥ 50 | 善线 | 正式善线剧情 |
| 20 ≤ 善恶值 < 50 | 善线指引 | 善线前置引导 |
| -50 < 善恶值 ≤ -20 | 恶线指引 | 恶线前置引导 |
| 善恶值 ≤ -50 | 恶线 | 正式恶线剧情 |
| -20 < 善恶值 < 20 | 无 | 中立区间，不触发任何剧情线 |

- 善/恶两条线互不干扰（独立判断）
- 同一条线内指引和正式互斥（不会同时触发）
- 不设冷却时间

## 条目结构

5 个世界书条目：

```
世界书/
  剧情控制.yaml        ← EJS 控制中枢，蓝灯始终激活
  善线指引.yaml        ← 剧情内容，禁用（仅被 include 时生效）
  恶线指引.yaml        ← 剧情内容，禁用（仅被 include 时生效）
  善线.yaml            ← 剧情内容，禁用（仅被 include 时生效）
  恶线.yaml            ← 剧情内容，禁用（仅被 include 时生效）
```

### 剧情控制条目

- 激活策略：蓝灯（始终激活）
- 插入位置：角色定义之前，顺序 100（紧跟世界观之后）
- 内容：纯 EJS 判断逻辑，通过 `<%- include('条目名') %>` 条件性引入对应剧情条目

### 四个剧情条目

- 激活策略：`启用: false`（不独立激活）
- 仅在被"剧情控制" include 时内容才会注入提示词
- 用户往各条目文件中手写剧情指引内容

## 数据流

```
MVU 变量更新 → user.善恶值 变化
                    ↓
下次 AI 生成时 → "剧情控制"条目（蓝灯）被处理
                    ↓
EJS 读取 getvar('stat_data.user.善恶值')
                    ↓
条件满足 → include 对应剧情条目 → 剧情内容注入提示词
条件不满足 → 不 include → 无剧情内容注入
```
